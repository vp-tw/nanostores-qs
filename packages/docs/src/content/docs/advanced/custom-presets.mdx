---
title: Custom Presets
description: Create reusable custom presets with createPreset
---

Use `createPreset` from `@vp-tw/nanostores-qs/presets` to build your own preset functions. A preset created with `createPreset` returns a function that accepts `{ optional }`, `{ default }`, or `{ array }` options — the same API as built-in presets.

```ts
import { createPreset } from "@vp-tw/nanostores-qs/presets";
```

## Percentage (bounded number)

```ts
const percentage = createPreset({
  decode: (value: unknown): number => {
    const n = Number(value);
    if (Number.isNaN(n) || n < 0 || n > 100) throw new Error("invalid percentage");
    return n;
  },
  defaultValue: 0,
  encode: (v) => String(v),
});

// percentage()                    — base (defaultValue: 0)
// percentage({ optional: true })  — no defaultValue
// percentage({ array: true })     — array variant
// percentage({ default: 50 })     — custom default
const progressStore = qsUtils.createSearchParamStore("progress", percentage());
```

## Comma-separated array

A single URL parameter representing an array (vs `{ array: true }` which uses multiple same-name parameters):

```ts
const commaSeparated = createPreset({
  decode: (value: unknown): string[] =>
    String(value)
      .split(",")
      .filter((s) => s.length > 0),
  defaultValue: [] as string[],
  encode: (v) => (v.length === 0 ? undefined : v.join(",")),
});

// URL: ?tags=react,typescript,nanostores
const tagsStore = qsUtils.createSearchParamStore("tags", commaSeparated());
```

## JSON object

Serialize complex objects to/from URL parameters:

```ts
const json = <T>(defaultValue: T) =>
  createPreset({
    decode: (value: unknown): T => {
      try {
        return JSON.parse(String(value)) as T;
      } catch {
        throw new Error("invalid JSON");
      }
    },
    defaultValue,
    encode: (v) => JSON.stringify(v),
  });

// URL: ?filter=%7B%22status%22%3A%22active%22%7D
const filterStore = qsUtils.createSearchParamStore("filter", json({ status: "active" })());
```

## decimal.js integration

```ts
import { createPreset } from "@vp-tw/nanostores-qs/presets";
import { Decimal } from "decimal.js";

const decimal = createPreset<Decimal>({
  decode: (v) => {
    const d = new Decimal(String(v));
    if (!d.isFinite()) throw new Error("invalid decimal");
    return d;
  },
  defaultValue: new Decimal(0),
  encode: (v) => v.toString(),
});

// decimal()                    — Decimal (default: 0)
// decimal({ optional: true })  — Decimal | undefined
// decimal({ array: true })     — Decimal[]
const priceStore = qsUtils.createSearchParamStore("price", decimal());
```

## Zod v4 integration

For schema validation, you can use a plain config object directly instead of `createPreset`:

```ts
import { z } from "zod";

const tabSchema = z.enum(["home", "settings", "profile"]);

const tabStore = qsUtils.createSearchParamStore("tab", {
  decode: (v) => tabSchema.parse(String(v)),
  defaultValue: "home" satisfies z.infer<typeof tabSchema>,
});
```

## Preset anatomy

A preset created by `createPreset` returns a **function** that accepts options:

- **No options** `()` — base config with `decode`, `defaultValue`, and `encode`
- **`{ optional: true }`** — omits `defaultValue`, making the type `T | undefined`
- **`{ array: true }`** — array variant using multiple same-name parameters
- **`{ default: value }`** — override `defaultValue` with a custom value
- **`{ array: true, maxItems: n }`** — array with a cap on items

```ts
const percentage = createPreset({
  decode: (value: unknown): number => {
    const n = Number(value);
    if (Number.isNaN(n) || n < 0 || n > 100) throw new Error("invalid percentage");
    return n;
  },
  defaultValue: 0,
  encode: (v) => String(v),
});

// Override the default from 0 to 50
const brightnessStore = qsUtils.createSearchParamStore("brightness", percentage({ default: 50 }));
```

Note: presets created with `createPreset` do not support `outOfRange` — that option is specific to the built-in `integer()` and `float()` presets. Handle range checking in your `decode` function instead.

Presets are **standalone** — they work with any `createQsUtils` instance, making them shareable across your app.
