When built-in presets don't cover your use case, you have two options:

1. **Plain config** — inline decode/encode for one-off usage
2. **`createPreset`** — reusable preset function with `{ optional }`, `{ default }`, `{ array }` options

## Zod v4

Extract enum values as `const` array, then create a Zod enum from it. The `const` array serves as both the Zod schema source and the type source — no need for `satisfies z.infer<...>`:

### Inline (plain config)

```ts
import { z } from "zod";

const tabOptions = ["home", "settings", "profile"] as const;
const tabEnum = z.enum(tabOptions);

const tabStore = qsUtils.createSearchParamStore("tab", {
  decode: (v) => tabEnum.parse(String(v)),
  defaultValue: tabOptions[0], // "home" — inferred as literal type
});
```

### Reusable preset

```ts
import { createPreset } from "@vp-tw/nanostores-qs/presets";
import { z } from "zod";

function zodEnum<const T extends readonly [string, ...Array<string>]>(values: T) {
  const schema = z.enum(values);
  return createPreset<T[number], T[0]>({
    decode: (v) => schema.parse(String(v)),
    defaultValue: values[0],
  });
}

const statusOptions = ["active", "inactive", "pending"] as const;
const statusPreset = zodEnum(statusOptions);

qsUtils.createSearchParamStore("status", statusPreset());
qsUtils.createSearchParamStore("status", statusPreset({ optional: true }));
qsUtils.createSearchParamStore("status", statusPreset({ array: true }));
```

## decimal.js

Use `resolve` when `$value` (for input binding) and `$resolved` (for calculation) need different types.

### Inline with resolve

```ts
import { Decimal } from "decimal.js";

const priceStore = qsUtils.createSearchParamStore("price", {
  decode: (v: unknown): string => {
    if (v == null) return "";
    try {
      const d = new Decimal(String(v));
      if (!d.isFinite()) return "";
      return d.toString();
    } catch {
      return "";
    }
  },
  defaultValue: "",
  encode: (v: string): string | undefined => (v === "" ? undefined : v),
  resolve: (v: string): Decimal => (v === "" ? new Decimal(0) : new Decimal(v)),
});

// priceStore.$value    — string (bindable to <input>)
// priceStore.$resolved — Decimal (for calculation)
```

### Reusable preset

```ts
import { createPreset } from "@vp-tw/nanostores-qs/presets";
import { Decimal } from "decimal.js";

const decimal = createPreset<Decimal>({
  decode: (v) => {
    const d = new Decimal(String(v));
    if (!d.isFinite()) throw new Error("invalid decimal");
    return d;
  },
  defaultValue: new Decimal(0),
  encode: (v) => v.toString(),
});

qsUtils.createSearchParamStore("tax", decimal());
qsUtils.createSearchParamStore("tax", decimal({ optional: true }));
qsUtils.createSearchParamStore("amounts", decimal({ array: true }));
```

## Plain config reference

### Config shape

| Property       | Type                            | Description                           |
| -------------- | ------------------------------- | ------------------------------------- |
| `decode`       | `(v: unknown) => T`             | Decode URL string to typed value      |
| `defaultValue` | `T`                             | Value when parameter is missing       |
| `encode`       | `(v: T) => string \| undefined` | Encode back to URL (undefined = omit) |
| `resolve`      | `(v: T) => R`                   | Optional: map `$value` → `$resolved`  |
| `isArray`      | `boolean`                       | Treat as array parameter              |

## `createPreset` options

A function returned by `createPreset` accepts:

| Option                         | Description                                                   |
| ------------------------------ | ------------------------------------------------------------- |
| `()`                           | Base — uses the decode, defaultValue, and encode you provided |
| `{ optional: true }`           | Omits `defaultValue`, type becomes `T \| undefined`           |
| `{ default: value }`           | Override `defaultValue`                                       |
| `{ array: true }`              | Array variant with per-item decode/encode                     |
| `{ array: true, maxItems: n }` | Array capped at `n` items                                     |

Note: `createPreset` does not support `outOfRange`, `min`/`max`, or `numInput` — those are specific to built-in `integer()` and `float()` presets. Bake constraint logic into your `decode` function instead.

## When to use what

| Scenario                                    | Approach                                   |
| ------------------------------------------- | ------------------------------------------ |
| Standard type (int, enum, boolean...)       | Built-in preset (e.g. `presets.integer()`) |
| Custom logic, one place                     | Plain config object                        |
| Custom logic, reused across stores          | `createPreset`                             |
| Need `$value` ≠ `$resolved` (input binding) | Add `resolve` to config                    |
