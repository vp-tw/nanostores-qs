When presets don't fit your use case, pass a **plain config object** directly to `createSearchParamStore`.

## Single-parameter config

```tsx
const numStore = qsUtils.createSearchParamStore("num", {
  decode: (v: unknown): string => {
    if (v == null) return "";
    const n = Number(v);
    if (Number.isNaN(n)) return "";
    return String(n);
  },
  defaultValue: "",
  encode: (v: string): string | undefined => (v === "" ? undefined : v),
  resolve: (v: string): number => (v === "" ? 0 : Number(v)),
});

// numStore.$value    â€” string (for input binding)
// numStore.$resolved â€” number (for business logic)
```

## Multi-parameter config

```tsx
const filters = qsUtils.createSearchParamsStore({
  search: { defaultValue: "" },
  category: { isArray: true },
  minPrice: {
    decode: (v: unknown) => Number(v),
    defaultValue: 0,
    encode: (v: number) => String(v),
  },
  maxPrice: {
    decode: (v: unknown) => Number(v),
    defaultValue: 1000,
    encode: (v: number) => String(v),
  },
  sortBy: { defaultValue: "newest" },
});
```

## Config shape

| Property       | Type                            | Description                                 |
| -------------- | ------------------------------- | ------------------------------------------- |
| `decode`       | `(v: unknown) => T`             | Decode URL string to typed value            |
| `defaultValue` | `T`                             | Value when parameter is missing             |
| `encode`       | `(v: T) => string \| undefined` | Encode value back to URL (undefined = omit) |
| `resolve`      | `(v: T) => R`                   | Optional: map `$value` â†’ `$resolved`        |
| `isArray`      | `boolean`                       | Treat as array parameter                    |

## When to use plain config

| Scenario                              | Approach                                            |
| ------------------------------------- | --------------------------------------------------- |
| Standard type (int, enum, boolean...) | Use a built-in preset (e.g. `presets.integer()`)    |
| Custom logic, used in one place       | Plain config object                                 |
| Custom logic, reused across stores    | Create a [custom preset](/advanced/custom-presets/) |
