title: Presets Reference

All presets are imported from `@vp-tw/nanostores-qs/presets`.

```ts
import { integer, enum as presetEnum, boolean } from "@vp-tw/nanostores-qs/presets";
```

## String presets

| Preset            | Value Type            | Default     | URL behavior              |
| ----------------- | --------------------- | ----------- | ------------------------- |
| `string`          | `string`              | `""`        | Omitted when empty        |
| `string.optional` | `string \| undefined` | `undefined` | Omitted when undefined    |
| `string.array`    | `string[]`            | `[]`        | Multiple same-name params |

## Number presets

| Preset             | Value Type            | Default     | URL behavior                    |
| ------------------ | --------------------- | ----------- | ------------------------------- |
| `integer`          | `number`              | `NaN`       | Integer, omitted when NaN       |
| `integer.optional` | `number \| undefined` | `undefined` | Integer, omitted when undefined |
| `integer.array`    | `number[]`            | `[]`        | Filters invalid values          |
| `float`            | `number`              | `NaN`       | Float, omitted when NaN         |
| `float.optional`   | `number \| undefined` | `undefined` | Float, omitted when undefined   |
| `float.array`      | `number[]`            | `[]`        | Filters invalid values          |

### Integer rounding variants

| Preset          | Description                 |
| --------------- | --------------------------- |
| `integer`       | Uses `Math.round` (default) |
| `integer.round` | Same as `integer`           |
| `integer.ceil`  | Uses `Math.ceil`            |
| `integer.floor` | Uses `Math.floor`           |
| `integer.parse` | Uses `parseInt` (truncates) |

### Float precision

```ts
float.fixed(2); // e.g., 3.14 — encodes with toFixed(2)
```

## Boolean presets

| Preset             | Value Type             | Default     | URL behavior                         |
| ------------------ | ---------------------- | ----------- | ------------------------------------ |
| `boolean`          | `boolean`              | `false`     | `"true"` present, omitted when false |
| `boolean.optional` | `boolean \| undefined` | `undefined` | `"true"`/`"false"` or omitted        |

## Enum presets

| Preset                   | Value Type               | Default       | URL behavior                    |
| ------------------------ | ------------------------ | ------------- | ------------------------------- |
| `enum(options)`          | `T[number]`              | First element | Falls back to first for invalid |
| `enum(options).optional` | `T[number] \| undefined` | `undefined`   | Undefined for invalid           |
| `enum(options).array`    | `T[number][]`            | `[]`          | Filters invalid values          |

### Enum usage

```ts
import { enum as presetEnum } from "@vp-tw/nanostores-qs/presets";

const sortOptions = ["newest", "oldest", "popular"] as const;

// Required enum — always one of the options
const sortStore = qsUtils.createSearchParamStore("sort", presetEnum(sortOptions));

// Optional enum — can be undefined
const filterStore = qsUtils.createSearchParamStore("filter", presetEnum(sortOptions).optional);

// Array of enums — filter invalid
const tagsStore = qsUtils.createSearchParamStore("tags", presetEnum(sortOptions).array);
```

## Date presets

| Preset          | Value Type          | Default         | URL behavior                     |
| --------------- | ------------------- | --------------- | -------------------------------- |
| `date`          | `Date`              | `new Date(NaN)` | ISO string, omitted when invalid |
| `date.optional` | `Date \| undefined` | `undefined`     | Undefined for invalid            |
| `date.array`    | `Date[]`            | `[]`            | Filters invalid dates            |

### Date usage

```ts
const startDate = qsUtils.createSearchParamStore("start", date.optional);
// URL: ?start=2024-01-15T00:00:00.000Z
```

## Additional presets

| Preset         | Value Type            | Default        | URL behavior          |
| -------------- | --------------------- | -------------- | --------------------- |
| `ymd`          | `string`              | `"0000-00-00"` | YYYY-MM-DD format     |
| `ymd.optional` | `string \| undefined` | `undefined`    | Undefined for invalid |
| `hms`          | `string`              | `"00:00:00"`   | HH:MM:SS format       |
| `hms.optional` | `string \| undefined` | `undefined`    | Undefined for invalid |

## Creating custom presets

Use `createPreset` to build your own preset with the same `.optional` and `.array` variants:

```ts
import { createPreset } from "@vp-tw/nanostores-qs/presets";

const myPreset = createPreset({
  decode: (v) => Number(v),
  defaultValue: 0,
  encode: (v) => String(v),
});

// myPreset           — base (defaultValue: 0)
// myPreset.optional  — no defaultValue
// myPreset.array     — array variant
```

See also [Custom Presets](/advanced/custom-presets/) for creating reusable presets with `defineSearchParam`.
